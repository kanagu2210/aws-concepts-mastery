# AWS Concept Mastery — Project Context

## What This Is

A Python pipeline that generates a 1000-concept AWS learning curriculum covering CCP, SAA, and SAP exams. Each concept goes through 6 stages to produce a narrated teaching script, an SVG whiteboard, and a rendered HTML/PDF learning page.

The project root is: `aws-concepts-mastery/`

```yaml

  slug: l2-vpc-peering-connection-model
  title: How VPC Peering Creates Direct Network Connections
  layer: 2                    # 1–6
  layer_name: Core Mechanisms
  spine_type: concept         # mental_model | concept | contrast | pattern | bridge

  domain: Networking
  aws_service: VPC
  concept_tier: core          # foundation | core | extension
  cognitive_role: model       # definition | contrast | model | application
  depth_stage: D3             # D1–D5 (controls length/depth)

  real_world_analogy: "..."
  confusion_buster: "..."

```

- L1 Foundations
- L2 Core Mechanisms
- L3 Service Mastery

- L6 Exam & Interview Bridges

## Pipeline Stages

### Stage 1 — Outline (`run_outline.py`)
Generates `input/concept_spines.yaml` from the outline prompt.
Run once to create the master spine list.

### Stage 2 — (Manual curation of spines)

### Stage 3 — Expand (`run.py`, expand stage)
- Input: one spine entry from `concept_spines.yaml`
- Prompt: `expand_prompt.md`
- Output: `output/01_concepts/layer_N/part_N/NNNN_slug.yaml`
- Expands the sparse spine into a full concept entry with all fields populated

### Stage 4 — Narrate (`run.py`, narrate stage)
- Input: expanded concept YAML
- Prompts: `narrate_plan_prompt.md` + `narrate_prompt.md`
- Output: `output/02_narrated/layer_N/part_N/NNNN_slug.md`
- Multipart: planning call splits includes into N parts, then N narration calls
- Produces a spoken teaching script with `[Pause: ...]` markers
- Narration rules:
  - Always uses one of four companies as analogy: Amazon, Netflix, Twitter/X, LinkedIn
  - Introduce company naturally: "Take Netflix, for example —" not "Think Netflix"
  - Opens with a session frame (what reader will understand by end)
  - Closes with a 2–4 sentence summary before the landing line
  - No interview or exam references in narration
  - No [Board:] markers — whiteboard decides its own panels independently

### Stage 4b — Whiteboard (`run_whiteboard.py`)
- Input: narrated `.md` file
- Prompt: `whiteboard_prompt.md`
- Output: `output/02b_whiteboards/layer_N/part_N/NNNN_slug.html`
- Standalone scrollable HTML page with progressive SVG diagrams
- Model reads the full narration and decides how many panels to draw
- Each panel shows cumulative board state (prior zones dimmed at opacity 0.72)
- max_tokens=32000 to avoid truncation
- Run independently: `python 02_pipeline/run_whiteboard.py --spine 78`

### Stage 4c — Curate (`run.py`, curate stage)
- Input: narrated `.md` file
- Prompt: `narration_curate_prompt.md`
- Output: updates `output/02_narrated/...` **in place**
- Repairs section boundary seams only (last 1–3 sentences + first 1–3 sentences at each `##` boundary)
- Hard constraint: ≤15% word delta from original
- Never touches [Pause:] markers, front matter, landing line, or middle of sections
- Appends CURATE_REPORT block (stripped before saving) for audit logging

### Stage 5 — Render (`run.py`, render stage)
- Input: narrated (curated) `.md` file
- Prompt: `render_prompt.md`
- Output: `output/03_rendered/layer_N/part_N/NNNN_slug.html`
- Produces a styled, self-contained HTML reading page
- Uses a three-level design system (Layout Skeleton × Reading Rhythm × Typographic Voice)
- PDF-first design (WeasyPrint compatible)
- [Pause:] markers rendered as styled visual elements

### Stage 6 — Publish (`run_publish.py`)
- Input: rendered HTML
- Output: `output/04_published/individual/...NNNN_slug.pdf`
- Uses WeasyPrint for HTML → PDF
- `--combine` merges per-layer PDFs; `--complete` builds the full curriculum PDF
- Run independently: `python 02_pipeline/run_publish.py --spine 78`

## Running the Pipeline

```bash
# Full pipeline for one spine
python 02_pipeline/run.py --spine 78

# Resume from a specific stage
python 02_pipeline/run.py --spine 78 --start-stage curate

# All spines, skip existing
python 02_pipeline/run.py --all --resume

# Range of spines
python 02_pipeline/run.py --from 71 --to 81

# Explicit list
python 02_pipeline/run.py --spines 71 72 78

# All spines in a layer
python 02_pipeline/run.py --layer 2

# Whiteboard (separate)
python 02_pipeline/run_whiteboard.py --spine 78
python 02_pipeline/run_whiteboard.py --all --resume

# Publish (separate)
python 02_pipeline/run_publish.py --spine 78
python 02_pipeline/run_publish.py --all

# Dashboard
python 02_pipeline/build_dashboard.py --open
```

**Stage selection flags:** `--start-stage expand | narrate | curate | render`

All scripts support: `--spine N`, `--spines N N N`, `--from N --to N`, `--layer N`, `--all`, `--resume`, `--overwrite`, `--dry-run`

## Key Files

### `pipeline_common.py`
Shared by all pipeline scripts. Contains:
- Path helpers: `concept_path()`, `narrated_path()`, `whiteboard_path()`, `rendered_path()`, `published_path()`
- Spine loaders: `_load_spines()`, `_get_spine()`
- `_select_spines(args)` — resolves all CLI selection args to a spine list
- `_strip_front_matter(md)` — splits YAML front matter from body
- `_part_subdir(spine_id)` — returns `layer_N/part_N` folder string

### `utils/api.py`
- `call_claude(user_content, model=None, max_tokens=None)` → `(text, in_tok, out_tok)`
- Auto-switches to streaming for `max_tokens > 8000` (avoids 10-min timeout)
- Retry logic: up to 5 attempts with exponential backoff on rate limits / 5xx

### `utils/config.py`
- Model: `claude-sonnet-4-20250514`
- Default max_tokens: `8000`
- All output paths configurable via `config.yaml` at project root

### `utils/usage_log.py`
- Appends TSV rows to `logs/usage_run.tsv`
- Columns: timestamp, stage, spine_id, slug, input_tokens, output_tokens, cost_usd, duration_s, status, notes
- Pricing: $3.00/M input tokens, $15.00/M output tokens (Sonnet 4)

## Dashboard (`build_dashboard.py`)

Generates `output/dashboard.html` — a self-contained browser dashboard.

Features:
- Three themes: Slate blue 🔵, Light 🌤, Warm cream 🌅 (persisted in localStorage)
- Three-level drill-down: Layer → Domain → AWS Service → Cards
- 3 cards per row, each showing: spine ID, status dot, type tag, exam badges, Read/Board buttons
- **3-pass revision tracking** (persisted in localStorage):
  - Pass 0: Not started
  - Pass 1: Read the narration (amber)
  - Pass 2: Study the whiteboard (blue)
  - Pass 3: Mastered (green)
- Revision state set from the slide-in viewer panel (80% width, slides from right)
- Filters: All / Rendered / With board / Pending / Pass 1/2/3 / To revise / Interview
- Footer metrics bar: stacked bar showing Pass 1/2/3 proportions + per-layer breakdown
- Settings panel (⚙): granular reset controls for each pass level

## Narration Style Rules (narrate_prompt.md)

- **Analogies**: One of four approved companies only — Amazon, Netflix, Twitter/X, LinkedIn
- **Introduction style**: "Take Netflix, for example —" or "Consider Amazon's situation:" (not "Think Netflix")
- **Structure**: Opens with a session frame → teaches → closing summary (2–4 sentences) → landing line
- **No exam/interview references** in narration content
- **[Pause: ...]** markers for directed reflection moments (not quizzes)
- **Depth stages**: D1 (600–900w) through D5 (2000–2800w)
- **Voice**: Spoken, contractions mandatory, fragments welcome, no banned phrases ("Let's explore", "In summary", "Key insight")

---

## Whiteboard Style Rules (whiteboard_prompt.md)

- Pure SVG — no ASCII, no JavaScript
- One SVG panel per natural visual moment (model decides, not markers)
- Panel count guide: D1–D2 = 2–3, D3 = 3–4, D4 = 4–5, D5 = 5–7
- Cumulative: each panel shows all prior zones (opacity 0.72) + new zone (opacity 1.0)
- Final panel: everything at full opacity
- Nine visual forms: Crisis Banner, Architecture Spine, Callout Annotation, Rule Pill, Step Chain, Comparison Table, Failure Scenario Map, Proportion Bars, Insight Line
- Colour palette by role (primary/control/traffic/failure/neutral/annotation)
- max_tokens=32000 to handle verbose SVG output
